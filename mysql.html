Module 1: Database Fundamentals

Duration: 1 week

What is a Database?

A database is an organized collection of structured information, or data, typically stored electronically in a computer system. Databases are managed by Database Management Systems (DBMS) which provide:

  • Data organization and structure
  • Data retrieval and manipulation capabilities
  • Data security and access control
  • Data integrity enforcement
  • Concurrent access management

Relational Database Concepts

Relational databases organize data into tables with rows and columns:

  • Tables: Represent entities (e.g., Customers, Orders)
  • Rows: Represent individual records
  • Columns: Represent attributes of the entity
  • Primary Keys: Unique identifiers for each row
  • Foreign Keys: Create relationships between tables

Creating Your First Database

Basic commands to create and manage databases:

-- Connect to MySQL server
mysql -u root -p

-- Create a new database
CREATE DATABASE company_db;

-- Switch to the new database
USE company_db;

-- Create a simple table
CREATE TABLE employees (
    id INT AUTO_INCREMENT PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    hire_date DATE NOT NULL,
    salary DECIMAL(10,2)
);

Module 2: SQL Essentials

Duration: 2 weeks

SQL Syntax and Structure

SQL (Structured Query Language) is used to manage relational databases. Key components:

  • DDL: Data Definition Language (CREATE, ALTER, DROP)
  • DML: Data Manipulation Language (SELECT, INSERT, UPDATE, DELETE)
  • DCL: Data Control Language (GRANT, REVOKE)
  • TCL: Transaction Control Language (COMMIT, ROLLBACK)

Data Types in MySQL

MySQL supports various data types:

-- Numeric types
INT, SMALLINT, TINYINT, MEDIUMINT, BIGINT
DECIMAL(10,2), FLOAT, DOUBLE

-- String types
CHAR(10), VARCHAR(255), TEXT, BLOB

-- Date and time types
DATE, TIME, DATETIME, TIMESTAMP, YEAR

-- Specialized types
ENUM('small','medium','large'), SET('red','green','blue')

CRUD Operations

Basic operations to manage data:

-- Create (INSERT)
INSERT INTO employees (first_name, last_name, email, hire_date, salary)
VALUES ('John', 'Doe', 'john.doe@example.com', '2023-01-15', 75000.00);

-- Read (SELECT)
SELECT * FROM employees;
SELECT first_name, last_name, salary FROM employees WHERE salary > 60000;

-- Update (UPDATE)
UPDATE employees
SET salary = 80000.00
WHERE last_name = 'Doe';

-- Delete (DELETE)
DELETE FROM employees
WHERE hire_date < '2020-01-01';

Sorting and Filtering Data

Organize and refine query results:

-- Basic sorting with ORDER BY
SELECT first_name, last_name, hire_date
FROM employees
ORDER BY hire_date DESC;

-- Filtering with WHERE
SELECT *
FROM employees
WHERE department_id = 5 AND salary > 50000;

-- Pattern matching with LIKE
SELECT *
FROM employees
WHERE last_name LIKE 'S%'; -- Starts with S

-- Range filtering with BETWEEN
SELECT *
FROM employees
WHERE hire_date BETWEEN '2022-01-01' AND '2022-12-31';

Module 3: Querying Data

Duration: 2 weeks

Advanced SELECT Statements

Go beyond basic queries with powerful techniques:

-- Using DISTINCT to remove duplicates
SELECT DISTINCT department_id FROM employees;

-- Limiting results with LIMIT
SELECT * FROM employees
ORDER BY salary DESC
LIMIT 10;

-- Using aliases for columns and tables
SELECT 
    e.first_name AS "First Name", 
    e.last_name AS "Last Name",
    d.department_name AS Department
FROM employees e
JOIN departments d ON e.department_id = d.department_id;

Table Joins

Combine data from multiple tables:

-- INNER JOIN (returns matching records)
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id;

-- LEFT JOIN (returns all records from left table)
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.department_id;

-- RIGHT JOIN (returns all records from right table)
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d ON e.department_id = d.department_id;

-- FULL OUTER JOIN (returns all records when match in left or right)
/* Note: MySQL doesn't support FULL OUTER JOIN directly, 
   but we can simulate with UNION */
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d ON e.department_id = d.department_id
UNION
SELECT e.first_name, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d ON e.department_id = d.department_id;

Module 4: Database Design

Duration: 1 week

Entity-Relationship Modeling

ER modeling is a conceptual design technique:

  • Entities: Real-world objects (e.g., Student, Course)
  • Attributes: Properties of entities (e.g., student_id, name)
  • Relationships: Associations between entities
  • Cardinality: One-to-One, One-to-Many, Many-to-Many

Normalization Techniques

Process to eliminate redundancy and dependency:

  • 1NF: Eliminate repeating groups, each cell contains atomic values
  • 2NF: Be in 1NF and remove partial dependencies
  • 3NF: Be in 2NF and remove transitive dependencies
  • BCNF: Boyce-Codd Normal Form - stricter version of 3NF

Stored Procedures and Functions

Reusable database logic:

-- Create a stored procedure
DELIMITER //
CREATE PROCEDURE GetEmployeeCount(IN dept_id INT, OUT emp_count INT)
BEGIN
    SELECT COUNT(*) INTO emp_count
    FROM employees
    WHERE department_id = dept_id;
END //
DELIMITER ;

-- Create a function
DELIMITER //
CREATE FUNCTION CalculateBonus(salary DECIMAL(10,2)) 
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    DECLARE bonus DECIMAL(10,2);
    SET bonus = salary * 0.15;
    RETURN bonus;
END //
DELIMITER ;
