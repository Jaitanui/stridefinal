Hyperlinks

Hyperlinks (or links) connect web pages. Created with the <a> tag:

<a href="https://example.com">Visit Example</a>

Link Attributes

Attribute Description Example
href Specifies the URL of the page href="page.html"
target Where to open the link target="_blank" (new tab)
rel Relationship between documents rel="nofollow"
download Prompts to save the linked URL download="filename"

Absolute vs Relative URLs

Absolute URLs

Complete web address including protocol and domain:

<a href="https://www.example.com/about.html">About Us</a>

Relative URLs

Point to resources relative to the current page:

<a href="contact.html">Contact</a>       <!-- Same directory -->
<a href="pages/about.html">About</a>  <!-- Subdirectory -->
<a href="../index.html">Home</a>      <!-- Parent directory -->
Tip: Use relative URLs for internal links to make your site more portable and easier to maintain.

When to Use Each

  • Absolute URLs - External resources, CDN links
  • Relative URLs - Internal site navigation, resources within your project

Image Formats

Different formats for different use cases:

JPEG (.jpg)

Best for: Photographs

Features: Lossy compression, no transparency

Quality: Adjustable

PNG (.png)

Best for: Graphics with transparency

Features: Lossless compression, alpha transparency

Quality: High (larger files)

GIF (.gif)

Best for: Simple animations

Features: Limited colors (256), supports animation

Quality: Low for photos

SVG (.svg)

Best for: Logos, icons, illustrations

Features: Vector format, scalable, small file size

Quality: Perfect at any size

Modern Tip: Use WebP format for better compression (smaller files with same quality).

Image Attributes

Essential attributes for the <img> tag:

<img 
  src="logo.png" 
  alt="Company Logo" 
  width="200" 
  height="100"
  loading="lazy"
  srcset="logo-1x.png 1x, logo-2x.png 2x"
>
Attribute Purpose Required
src Path to the image file Yes
alt Alternative text for accessibility Yes
width/height Dimensions (prevents layout shifts) Recommended
loading Loading behavior (lazy/eager) Optional
srcset Responsive images for different resolutions Optional
Accessibility Tip: Always provide meaningful alt text. For decorative images, use alt="".

Favicons

Small icons displayed in browser tabs and bookmarks.

Basic Implementation

<head>
  <link rel="icon" href="favicon.ico" type="image/x-icon">
  <link rel="apple-touch-icon" href="apple-touch-icon.png">
</head>

Modern Approach

<head>
  <link rel="icon" href="favicon.svg" type="image/svg+xml">
  <link rel="icon" href="favicon.png" type="image/png">
  <link rel="apple-touch-icon" href="apple-touch-icon.png">
  <link rel="manifest" href="site.webmanifest">
</head>
